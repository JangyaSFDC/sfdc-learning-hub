SOQL Best Practices & Query Optimization: Write Faster, Smarter Queries

SH
SFDC Hub Team
Published on March 24, 2026

Why SOQL Optimization Matters

Salesforce Object Query Language (SOQL) is the backbone of data retrieval in the Salesforce platform. Whether you're building Apex classes, Lightning components, or reports, your SOQL queries directly impact performance. Poorly written queries can hit governor limits, slow down user experience, and even cause runtime errors in production.

A single poorly optimized SOQL query inside a loop can bring an entire transaction to its knees. Mastering SOQL best practices is non-negotiable for any serious Salesforce developer.

1. Always Use Selective Queries

A selective query is one that uses indexed fields in the WHERE clause. Salesforce maintains indexes on certain fields by default:

  • Id — Always indexed
  • Name — Always indexed
  • OwnerId — Always indexed
  • CreatedDate — Always indexed
  • SystemModstamp — Always indexed
  • RecordTypeId — Always indexed
  • Lookup fields — Always indexed
  • Custom fields marked as External ID or Unique — Custom indexed
// GOOD - Uses indexed field (Id)
List<Account> accounts = [SELECT Id, Name FROM Account WHERE Id IN :accountIds];

// BAD - Uses non-indexed custom field without filter
List<Account> accounts = [SELECT Id, Name FROM Account WHERE Custom_Field__c = 'value'];

If you frequently query on a custom field, request a custom index from Salesforce Support to improve performance.

2. Never Put SOQL Inside Loops

This is the #1 cause of governor limit exceptions. Salesforce allows only 100 SOQL queries per synchronous transaction. Putting a query inside a for-loop burns through this limit instantly.

// BAD - SOQL inside loop (will hit 100 query limit)
for (Account acc : accountList) {
    List<Contact> contacts = [SELECT Id, Name FROM Contact WHERE AccountId = :acc.Id];
    // process contacts
}

// GOOD - Query once, then process
Map<Id, List<Contact>> contactsByAccount = new Map<Id, List<Contact>>();
for (Contact con : [SELECT Id, Name, AccountId FROM Contact WHERE AccountId IN :accountIds]) {
    if (!contactsByAccount.containsKey(con.AccountId)) {
        contactsByAccount.put(con.AccountId, new List<Contact>());
    }
    contactsByAccount.get(con.AccountId).add(con);
}

// Now loop through accounts and use the map
for (Account acc : accountList) {
    List<Contact> relatedContacts = contactsByAccount.get(acc.Id);
    // process related contacts
}

3. Query Only the Fields You Need

Every field you add to your SELECT clause adds to heap size and processing time. Only retrieve what you actually use.

// BAD - Selecting everything (and you can't do SELECT * in SOQL anyway)
List<Account> accounts = [SELECT Id, Name, Phone, Website, Industry, 
    BillingStreet, BillingCity, BillingState, BillingPostalCode,
    ShippingStreet, ShippingCity, ShippingState, ShippingPostalCode,
    Description, NumberOfEmployees, AnnualRevenue
    FROM Account WHERE Id IN :accountIds];

// GOOD - Only select what you need
List<Account> accounts = [SELECT Id, Name, Industry FROM Account WHERE Id IN :accountIds];

4. Use LIMIT to Control Record Count

When you don't need all records, use LIMIT to cap results. This is especially useful in batch processes, list views, and test methods.

// Get only the 10 most recent Opportunities
List<Opportunity> recentOpps = [
    SELECT Id, Name, Amount, CloseDate 
    FROM Opportunity 
    WHERE StageName = 'Closed Won'
    ORDER BY CloseDate DESC 
    LIMIT 10
];

5. Use Relationship Queries Instead of Multiple Queries

SOQL supports parent-to-child and child-to-parent relationship queries. Use them to reduce the total number of queries.

// PARENT-TO-CHILD (subquery) - Get accounts with their contacts in ONE query
List<Account> accountsWithContacts = [
    SELECT Id, Name, 
        (SELECT Id, FirstName, LastName, Email FROM Contacts)
    FROM Account 
    WHERE Id IN :accountIds
];

// CHILD-TO-PARENT - Get contact with account info in ONE query
List<Contact> contacts = [
    SELECT Id, FirstName, LastName, Account.Name, Account.Industry
    FROM Contact 
    WHERE AccountId IN :accountIds
];

6. Use FOR UPDATE for Record Locking

When you need to update records and want to prevent concurrent modifications, use FOR UPDATE to lock records.

// Lock records to prevent concurrent updates
List<Account> accounts = [
    SELECT Id, Name, AnnualRevenue 
    FROM Account 
    WHERE Id = :accountId 
    FOR UPDATE
];

Use FOR UPDATE sparingly — it locks records and can cause other transactions to wait, leading to timeout errors if overused.

7. Aggregate Queries for Reporting

Instead of fetching all records and calculating in Apex, use SOQL aggregate functions:

// Count records by industry
List<AggregateResult> results = [
    SELECT Industry, COUNT(Id) total, SUM(AnnualRevenue) revenue
    FROM Account 
    WHERE Industry != null 
    GROUP BY Industry
    HAVING COUNT(Id) > 5
    ORDER BY COUNT(Id) DESC
];

for (AggregateResult ar : results) {
    System.debug(ar.get('Industry') + ': ' + ar.get('total') + ' accounts, $' + ar.get('revenue'));
}

8. SOQL Governor Limits Cheat Sheet

Keep these limits in mind when writing queries:

  • 100 — Maximum SOQL queries per synchronous transaction
  • 200 — Maximum SOQL queries per asynchronous transaction
  • 50,000 — Maximum records retrieved by SOQL queries per transaction
  • 2,000 — Maximum records returned by a SOQL subquery
  • 20 — Maximum relationship subqueries in a single query
  • 35 — Maximum GROUP BY expressions

9. Use Bind Variables for Security

Always use bind variables (:variable) instead of string concatenation to prevent SOQL injection attacks:

// BAD - Vulnerable to SOQL injection
String userInput = 'test';
String query = 'SELECT Id FROM Account WHERE Name = \'' + userInput + '\'';
List<Account> accounts = Database.query(query);

// GOOD - Safe with bind variable
String userInput = 'test';
List<Account> accounts = [SELECT Id FROM Account WHERE Name = :userInput];

// GOOD - If dynamic SOQL is needed, use escapeSingleQuotes
String userInput = String.escapeSingleQuotes('test');
String query = 'SELECT Id FROM Account WHERE Name = \'' + userInput + '\'';
List<Account> accounts = Database.query(query);

Summary

Writing efficient SOQL queries is a foundational skill for every Salesforce developer. Remember these key principles: use selective queries with indexed fields, never put SOQL in loops, query only the fields you need, leverage relationship queries, and always use bind variables for security. Following these best practices will help you build applications that are performant, scalable, and governor-limit-friendly.
